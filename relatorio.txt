Etapa 1 - Preparação do ambiente
Após a instalação dos pré-requisitos para rodar o Kathará, foi criada a pasta lab, que será responsável pela organização do ambiente. Nela, foram criados os arquivos .startup de cada máquina, bem como suas respectivas pastas.

Foram definidos os endereços IP (para aqueles que possuem endereços estático) para os dispositivos em cada interface de rede.

Foi feita a configuração de roteamento dos dispositivos, atribuindo todas as rotas possíveis para cada máquina, a fim de estabelecer comunicação entre todas as máquinas.

Topologia estando pronta, realizou-se testes com o comando ping para verificar a comunicação entre os dispositivos. Teste concluído com sucesso!

Após a criação da topologia, decidi realizar a configuração NAT e configurar o modo bridge no Roteador 1. Ademais, adicionei rotas default para o Roteador 2 referenciando o IP do Roteador 1, que já está com acesso à internet.

Feita a implementação do NAT, iniciei a configuração do servidor DNS. Defini o seu servidor DNS padrão para 8.8.8.8 (servidor do Google), logo em seguida instalei atualizações necessárias e então instalei os programas necessários para iniciar o serviço DNS na máquina. Configurei os arquivos de zona e zona reversa, bem como defini os arquivos de configurações "named.conf.local" e "named.conf.options" para que obtenha sucesso na inicialização do serviço. Persisti as configurações criando os arquivos necessários dentro do diretório /etc dentro da pasta do servidor DNS no diretório do projeto Kathará.

Com o servidor DNS funcionando, decidi iniciar a configuração do servidor DHCP. Pra isso, foi necessário definir o endereço IP do servidor DNS previamente criado (10.0.0.253) dentro do arquivo 'resolv.conf'. Feito isso, adicionei as rotas e então tive que rodar um comando para esperar dois minutos até que os próximos comandos sejas executados, pois sem esse comando a máquina tenta acessar a o servidor DNS para resolver os domínios para atualização sem o servidor DNS estar pronto. Passado o tempo necessário, a máquina instala atualizações e os pacotes necessários para inicialização do servidor DHCP. Também foi necessário criar os arquivos '/default/isc-dhcp-server' e 'dhcp/dhcpd.conf' dentro da pasta 'dhcp/etc' para que as configurações DHCP sejam persistidas. Feitas todas essas configurações, o servidor DNS estava pronto, sendo apenas necessário rodar o comando para que o cliente DHCP (PC) faça a requisição de um endereço IP; e isso foi feito através do comando 'dhclient' dentro do arquivo de inicialização do PC.

Após a configuração DNS e DHCP, todos os dispositivos já estavam com seus respectivos endereços IP e com acesso à internet. Decidi então criar o servidor FTP. A configuração foi relativamente simples, sendo apenas necessário, como aconteceu na configuração DHCP, executar o comando para o tempo de espera até que o servidor DNS fique pronto e, então, instalar atualizações necessárias, bem como o pacote 'proftpd'; que é o serviço FTP propriamente dito. Após, tive que definir os três usuários para autenticação, bem como suas senhas. Ainda precisei criar o arquivo 'proftpd.conf' dentro da pasta 'ftp/etc/proftpd', que, como feito nos servidores anteriores, foi necessário para que a configuração do servidor seja persistida. Dentro desse arquivo, alterei algumas linhas para que pudesse habilitar o login anônimo, bem como o "alias" do usuário anônimo ('anonymous' e 'ftp'). Ainda foi necessário criar a pasta '/files', dentro do diretório '/ftp' para que não houvesse erro ao tentar fazer login como usuário anônimo.
